# VR Endless Forest — Specification Dashboard

Popperian refutation-aware confidence tracking for the VR Forest specification suite. Specs are treated as conjectures — confidence comes from surviving structured attempts to break them.

Adapted for a client-side WebXR project using a Popperian refutation model where confidence is earned by surviving structured attempts to break each spec.

## Quick Start

```bash
# Generate the dashboard (one-shot)
bash specs/dashboard/build.sh

# Open in browser
open specs/dashboard/index.html

# Or serve with live editing API
python3 specs/dashboard/server.py --port 8090
# Visit http://localhost:8090/
```

## Confidence Model

Six evidence columns, each with 3 signals. Overall confidence is the weighted average of column scores.

| Column | Label | Signals | Weight |
|--------|-------|---------|--------|
| `reviewed` | Survived Peer Review | `review1`, `review2`, `expert` | 0.15 |
| `test_spec` | Survived Test Vectors | `spec_vectors`, `json_vectors`, `adversarial` | 0.20 |
| `built` | Survived Implementation | `partial`, `complete`, `optimised` | 0.20 |
| `testing` | Survived Automated Testing | `unit`, `automated`, `visual` | 0.20 |
| `integrated` | Survived Integration | `component`, `cross_system`, `full_scene` | 0.15 |
| `field` | Survived VR Testing | `desktop`, `quest3`, `sustained` | 0.10 |

### Adaptations from R2

- **`testing.visual`** replaces `testing.human` — visual regression is more relevant than manual testing for a 3D rendering project
- **`integrated.full_scene`** replaces `integrated.system` — integration means all systems running together in one scene
- **`field.desktop`** + **`field.quest3`** + **`field.sustained`** replace lab/pilot/production — the two deployment targets are desktop browser and Quest 3 headset, plus sustained play (30+ min, no frame drops, no memory leaks)

## Decay & Penalties

| Event | Effect |
|-------|--------|
| Dependency version changed | -0.10 overall, -0.15 on testing/integrated/field cells |
| Open issue | -0.03 per issue |
| Spec version bump | Resets confidence to 0.15 (exists bonus only) |
| Stale evidence (spec changed since review) | Evidence decays to 40% weight |
| Low-confidence dependency | Up to -30% propagated reduction |

## Colour Bands

| Band | Confidence | Meaning |
|------|-----------|---------|
| Red | < 0.30 | Untested conjecture |
| Yellow | 0.30 – 0.59 | Emerging evidence |
| Green | 0.60 – 0.84 | Survived multiple refutations |
| Full | >= 0.85 | Battle-tested |

## Updating Evidence

The normal workflow is conversational — tell [Claude Code](https://claude.ai/claude-code) what you've verified and it updates the dashboard for you:

> "all specs have survived VR testing on desktop and Quest 3"
> "VF-TERRAIN has unit tests passing now"
> "I've reviewed all the specs"

Claude runs the `review.py` CLI commands, rebuilds the dashboard, and pushes. You can also run the CLI directly:

## CLI Usage

```bash
# Set evidence
python3 specs/dashboard/review.py VF-TERRAIN --set built.optimised=true
python3 specs/dashboard/review.py VF-TERRAIN --set testing.unit=true

# Record a review
python3 specs/dashboard/review.py VF-TERRAIN \
  --reviewer "Roy Davies" --status approved \
  --type implementation_test --severity thorough \
  --note "Full implementation verified against spec"

# Add an issue
python3 specs/dashboard/review.py VF-WATER --issue "Shore foam grid spacing inconsistent with spec"

# Resolve an issue
python3 specs/dashboard/review.py VF-WATER --resolve-issue 0

# Show current evidence
python3 specs/dashboard/review.py VF-TERRAIN --show
```

## File Structure

```
dashboard/
├── build.sh              # One-shot: parse → calculate → generate
├── watch.sh              # Poll for changes, auto-rebuild
├── parse_specs.py        # Extract VF-* metadata from markdown
├── calculate.py          # Popperian confidence scoring
├── spec_state.py         # Shared state helpers, staleness, propagation
├── generate.py           # Emit index.html from state + confidence
├── review.py             # CLI: approve specs, set evidence, manage issues
├── server.py             # HTTP server for live dashboard editing
├── app.js                # Interactive dashboard JS (embedded in index.html)
├── index.html            # Generated — do not edit directly
└── spec-meta/
    ├── dependencies.json     # Auto-generated from spec parsing
    ├── confidence.json       # Auto-generated from calculate
    ├── descriptions.json     # Spec conjecture descriptions for popups
    ├── reviewer_roster.json  # Reviewer assignments
    └── reviews/
        └── VF-*.json         # Per-spec evidence + review history
```

## Rules

1. **Never hand-edit `index.html`** — it is generated by `generate.py`
2. **Confidence is earned, not granted** — set evidence via `review.py` or the server API
3. **Dependency changes propagate** — when a dependency spec changes, downstream confidence decays
4. **Stale evidence fades** — evidence gathered before the spec was last modified decays to 40%
